# محاكي نظام إدارة المكتبة (Library Simulator)

مشروع برمي متكامل تم تطويره كجزء من مقرر مادة علوم الحاسب CSC 113 (البرمجة كائنية التوجه - Object-Oriented Programming). يحاكي المشروع آلية عمل الأنظمة الواقعية لإدارة المكتبات باستخدام لغة Java، ويعتمد بشكل أساسي على تطبيق المفاهيم البرمجية المتقدمة التي تم دراستها في المقرر لتنظيم وإدارة البيانات المتعلقة بالكتب، المؤلفين، المستخدمين، وعمليات البحث والإعارة.

## المفاهيم البرمجية المطبقة (OOP and Architecture)
تم تصميم المشروع بناءً على المعايير الأكاديمية لمادة CSC 113 من خلال تقسيم النظام إلى فئات (Classes) وواجهات (Interfaces) مترابطة:
- التغليف والكبسلة (Encapsulation): لحماية بيانات الكتب والمستخدمين والمؤلفين عبر الخصائص الخاصة والدوال (Getters and Setters).
- الواجهات والتجريد (Interfaces and Abstraction): استخدام الـ SearchInterface لتوفير معايير مرنة ومنفصلة لعمليات البحث داخل الكتالوج.
- العلاقات بين الكائنات (Object Relationships): تطبيق مفاهيم الارتباط والتركيب بين الكائنات مثل ربط الكتب بالمؤلفين، وإدارة بطاقات المستخدمين وعمليات الاستعارة.

## الهيكل البرمجي للمشروع (Project Structure)
- ملفات `Book` و `Author`: لتمثيل بيانات الكتب وربطها بالمؤلفين وتفاصيل النشر.
- ملفات `Catalog` و `SearchInterface`: يمثل مستودع الكتب الداخلي المسؤول عن تنظيم الفهارس وإجراء عمليات البحث المتقدمة.
- ملفات `User` و `Card`: لإدارة حسابات المشتركين في المكتبة، صلاحياتهم، وبيانات بطاقات العضوية الخاصة بهم.


## التقنيات المستخدمة (Tech Stack)
- اللغة الأساسية: Java (تطبيق مفاهيم الـ OOP وهياكل البيانات الأساسية للمقرر).
- بيئة التطوير: متوافق مع كافة بيئات عمل الجافا (مثل Eclipse، IntelliJ، أو NetBeans).

## كيف تشغل المشروع؟ (How to Run)
1. قم بعمل Clone أو تحميل للمستودع على جهازك.
2. افتح المشروع باستخدام أي بيئة تطوير تدعم الجافا (IDE).
3. قم بتشغيل الملف الرئيسي (الذي يحتوي على دالة main) لبدء المحاكاة واختبار النظام.
